DotTalk++ Memo Features (Alpha, sidecar .dtx)

NEW/UPDATED VERBS
-----------------
TYPE <field> [TO <path>]
    Show a memo field in full (paged). If TO <path> is given, write to that file.

REPLACE <field> WITH "text..."
REPLACE <field> WITH FILE "<path>"
    Set/replace memo content directly from a literal or a file path.

SET MEMO WIDTH <n>
    Controls how many characters of memo are shown in LIST (default 60).

SET MEMO EDITOR "<exe path>"
    External editor to launch for EDIT <field>. If not set, use inline multi-line
    input terminated by a single dot '.' on its own line.

SET MEMO TRUNCATE ON|OFF
    If OFF, DISPLAY shows full memo even if long; LIST still truncates.

MEMOCHECK [REPAIR]
    Validate memo sidecar integrity (CRC/length). With REPAIR, attempt to rebuild
    id map and reclaim orphans.

MEMOSTATS
    Print memo store stats (Used, Free, Objects, NextId, LargestHole if available).

BEHAVIOR
--------
- LIST truncates memo fields to MEMO WIDTH and shows "(len=XXXX)" when longer.
- DISPLAY shows full unless MEMO TRUNCATE is ON and width set.
- Memos are not indexable in this release.
- PACK compacts the sidecar and rewrites id-map; ZAP truncates both DBF and sidecar.
- Deleted rows' memos are reclaimed on PACK (not immediately on DELETE).

ERRORS
------
- Missing sidecar when memo fields exist => table open fails with a clear message.
- CRC mismatch => object flagged; MEMOCHECK reports; use --ignore-crc option if provided.
