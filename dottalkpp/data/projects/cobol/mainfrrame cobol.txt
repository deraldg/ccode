# 📦 2. Exporting DBF Records as COBOL‑Friendly Fixed Files  
Your DBF_64 record layout already contains:

- field name  
- type  
- length  
- decimals  
- offset  

COBOL needs:

- PIC clauses  
- exact byte lengths  
- no delimiters  
- no null terminators  

So your export routine should produce a file like:

```
CUSTOMERS.FLT
ORDERS.FLT
INVENTORY.FLT
```

Each line = one DBF record, padded to full length.

### Example C++ export loop
```cpp
for (auto &rec : dbf.records()) {
    std::string line;
    line.reserve(dbf.record_length);

    for (auto &f : dbf.fields) {
        std::string v = rec.get(f);
        v.resize(f.length, ' ');  // pad to fixed length
        line += v;
    }

    out << line << "\n";
}
```

This gives COBOL a perfect flat file.

# 🔄 4. COBOL → DBF_64 (Importing Back)  
COBOL writes fixed‑length records too.

So your import routine is the mirror:

```cpp
std::string line;
while (std::getline(in, line)) {
    DBFRecord rec;
    size_t pos = 0;

    for (auto &f : dbf.fields) {
        std::string v = line.substr(pos, f.length);
        rec.set(f, v);
        pos += f.length;
    }

    dbf.append(rec);
}
```

This gives you a full round‑trip.
