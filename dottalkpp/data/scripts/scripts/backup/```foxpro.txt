```foxpro
* alltypes_5gb_memo.prg
* ================================================
* Creates a table with ONE field of EVERY major xBase/VFP data type
* Appends 1 record, populates every field
* Fills the Memo field with ~5 GB of repeating prose
* (chunked so it doesn't explode RAM)
* ================================================

CLEAR
SET TALK OFF
SET SAFETY OFF

* ---------- 1. CREATE TABLE WITH ALL DATA TYPES ----------
CREATE TABLE alltypes FREE ( ;
    c_char      C(100),     ;
    n_numeric   N(15,5),    ;
    f_float     F(15,5),    ;
    d_date      D,          ;
    t_datetime  T,          ;
    l_logical   L,          ;
    m_memo      M,          ;   && this will become 5 GB
    i_integer   I,          ;
    b_double    B,          ;
    y_currency  Y,          ;
    v_varchar   V(200),     ;
    q_varbinary Q(200)      ;
)

? "Table created with all data types: " + DBF()
USE alltypes EXCLUSIVE

* ---------- 2. APPEND BLANK RECORD ----------
APPEND BLANK
? "Record appended (RecNo = " + TRANSFORM(RECNO()) + ")"

* ---------- 3. POPULATE NORMAL FIELDS ----------
REPLACE c_char     WITH "Hello from xBase test!"
REPLACE n_numeric  WITH 12345.67890
REPLACE f_float    WITH 9876.54321
REPLACE d_date     WITH DATE()
REPLACE t_datetime WITH DATETIME()
REPLACE l_logical  WITH .T.
REPLACE i_integer  WITH 2147483647
REPLACE b_double   WITH 1234567890.12345
REPLACE y_currency WITH 999999.99
REPLACE v_varchar  WITH "This is variable-length text up to 200 chars"
REPLACE q_varbinary WITH CHR(0xDE) + CHR(0xAD) + CHR(0xBE) + CHR(0xEF)

? "Normal fields populated."

* ---------- 4. CREATE 5 GB MEMO WITH PROSE (CHUNKED) ----------
LOCAL lcProseChunk, lnTargetGB, lnTargetBytes, lnWritten, lnChunkSize

lnTargetGB    = 5.0
lnTargetBytes = INT(lnTargetGB * 1024 * 1024 * 1024)
lnChunkSize   = 10 * 1024 * 1024          && 10 MB per chunk (safe for RAM)
lnWritten     = 0

* Sample prose sentence (≈100 bytes)
lcProseChunk = REPLICATE("The quick brown fox jumps over the lazy dog. " + ;
                         "This is filler prose to demonstrate a massive memo field. " + ;
                         "xBase engines can store gigabytes in .FPT files. ", ;
                         INT(lnChunkSize / 100))

? "Starting 5 GB memo fill (chunked - low RAM usage)..."

DO WHILE lnWritten < lnTargetBytes
    REPLACE m_memo WITH m_memo + lcProseChunk   && appends to existing memo

    lnWritten = lnWritten + LEN(lcProseChunk)
    IF MOD(lnWritten, 100*1024*1024) = 0   && every 100 MB
        ? "   Memo size now: " + TRANSFORM(lnWritten / (1024^3), "999.99") + " GB"
    ENDIF
ENDDO

* Final trim if we overshot (optional)
IF LEN(m_memo) > lnTargetBytes
    REPLACE m_memo WITH LEFT(m_memo, lnTargetBytes)
ENDIF

? "✅ DONE! Memo field now contains ≈" + TRANSFORM(LEN(m_memo)/(1024^3), "999.99") + " GB of prose."
? "Table: " + DBF()
? "Memo file: " + DBF("FPT")
? "Record count: " + TRANSFORM(RECCOUNT())

USE
SET TALK ON
```

### How to run it
1. Save as `alltypes_5gb_memo.prg`
2. In your xBase CLI/shell:
   ```
   DO alltypes_5gb_memo.prg
   ```
3. Or `SCHEMAS OPEN .` then run the PRG.

### Performance notes (important!)
- **Time**: ~5–15 minutes on a fast SSD (depends on your engine's .FPT write speed).
- **RAM**: Only ~10–20 MB peak (thanks to chunked `REPLACE m_memo WITH m_memo + ...`).
- **Disk**: Will create `alltypes.dbf` (~1 KB) + `alltypes.fpt` (≈5.0 GB).
- **Test first**: Change `lnTargetGB = 0.1` (100 MB) to verify everything works quickly.
