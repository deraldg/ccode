## Parsing rules to lock now

1. `VAR_OK = N` means reject as a DotScript variable name.
2. `HARD = Y` means always reserved in statement grammar.
3. `SOFT = Y` means only special when the active command grammar expects it.
4. `EXPR = Y` means expression parser owns it.
5. Quoted strings bypass all reserved-word checks.

## First rejection set for `VAR`

These should fail immediately:

```text
VAR if = 1
VAR and = 1
VAR true = 1
VAR all = 1
VAR set = 1
VAR off = 1
VAR next = 1
```

## First DotScript shakedown

```text
* dotscript_reserved_shakedown_01.dts

var alpha = 1

    ? "IF AND parsed"

* these should fail cleanly
var if = 1

var true = 1

* these should remain data

* these should fail unless later supported
```

## Immediate next step

Build three small policy functions:

* `is_hard_reserved(token)`
* `is_soft_reserved(token)`
* `is_valid_dotscript_var_name(token)`

Then wire the `VAR` validator first. That gives you the quickest safety gain.
