# ⭐ **DotTalk++ / Microsoft 365 Integration — Milestone 1 Implementation Plan**  
### *Primitive Interchange Layer — Phase 1 (Foundational Build)*

This milestone focuses on **infrastructure**, **path resolution**, **file classification**, and **basic export/import scaffolding**.  
No business logic. No schema enforcement. No heavy parsing.  
Just the skeleton that everything else will hang on.

---

# **🎯 Milestone 1 Goals**
By the end of Milestone 1, DotTalk++ will be able to:

- Detect OneDrive  
- Create the `DotTalk_Exchange` folder structure  
- Build correct paths  
- Generate timestamps  
- Classify inbound/outbound files  
- Export placeholder CSV/JSON/TXT files  
- Scan inbound folder and list files  
- Archive files into `archive/YYYY/MM/`  

This gives you a **working pipeline** even before real data flows through it.

---

# **📌 Milestone 1 Deliverables**

## **1. OneDrive Root Detection**
Implement the detection logic:

- Check environment variables  
- Check known Windows paths  
- Support override via `DOTTALK_ONEDRIVE_ROOT`  
- Cache result  

**Output:**  
`oneDriveRoot` string stored in global context.

---

## **2. Folder Initialization**
Create the full directory tree:

```
DotTalk_Exchange/
   inbound/
   outbound/
   archive/
```

Rules:

- Create if missing  
- Validate read/write permissions  
- Fail with `ERR_PATH_CREATE` if not possible  

---

## **3. Timestamp Utilities**
Implement:

- `make_current_timestamp_utc()`  
- ISO‑8601 generator  
- Filename‑safe timestamp generator  

This is used by all exports.

---

## **4. Path Builders**
Implement:

- `build_path(subfolder, filename)`  
- `classify_file(full_path, file_name)`  

Classification must detect:

- table CSV  
- record JSON  
- notes TXT  
- inbound vs outbound  

This is the backbone of the entire interchange layer.

---

## **5. Outbound Export Stubs**
Implement **minimal** versions of:

- `export_table_to_csv`  
- `export_record_to_json`  
- `export_notes_to_txt`  

For Milestone 1:

- Write placeholder content  
- Confirm file creation  
- Confirm naming conventions  
- Confirm correct folder placement  

Real data comes in Milestone 2.

---

## **6. Inbound Scan**
Implement:

- `scan_inbound_folder()`  

This should:

- list files  
- classify them  
- return `InboundScanResult`  

No parsing yet — just detection.

---

## **7. Archive Mechanism**
Implement:

- `archive_file(descriptor, out_path)`  

Rules:

- Create `archive/YYYY/MM/`  
- Move file  
- Return archived path  

This completes the round‑trip skeleton.
